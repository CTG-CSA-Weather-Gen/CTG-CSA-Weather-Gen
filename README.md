This repository contains the official PyTorch implementation of the **CTG-CSA** model from the paper:  
> *"High-Fidelity Synthetic Meteorological Year Generation Model Integrating Physical Priors and Dual Attention"*  
> (Currently under peer review / Accepted in *Discover Applied Sciences*).

## 📖 Overview

The CTG-CSA framework proposes a Physics-Guided Machine Learning (PGML) paradigm to generate High-Fidelity Synthetic Meteorological Years (HF-SMY). It effectively overcomes the limitations of pure data-driven models (e.g., weak extrapolation, mode collapse) by extending the Conditional Tabular GAN (CTGAN) with:
1. **Channel Attention (CA)**: Guided by atmospheric thermodynamic coupling priors (e.g., the Clausius-Clapeyron relation).
2. **Spatial Attention (SA)**: Guided by geographical priors derived from the Variance Ratio Criterion (VRC).
3. **Physics-Informed WGAN-GP**: An adversarial training framework seamlessly integrated with multidimensional physical constraint losses.

## 🛠️ Custom Dataset Integration

To train the CTG-CSA model on your own localized weather data, please follow these strict formatting protocols:
4D Tensor Format: Prepare your input data in the shape of (Samples, Channels/Variables, Temporal Sequence, Spatial Stations). For example, (32, 5, 24, 20) represents 32 batches, 5 variables, 24-hour sequences, across 20 stations.
Variable Ordering (Crucial): The deterministic physical bounding layer in Generator applies specific activation functions to prevent non-physical anomalies (e.g., negative wind speed, >100% humidity). Your variables must be ordered as:
Index 0: Near-surface air temperature
Index 1: Relative humidity (Strictly clipped to [0, 100]%)
Index 2: Wind speed (Bounded by ReLU)
Index 3: Total solar irradiance (Bounded by ReLU)
Index 4: Atmospheric pressure
Geographical Priors: Recalculate the spatial physical prior matrix (

## 🏢 Downstream Application: Building Energy Simulation
As demonstrated in the paper (Table 1), the HF-SMY data generated by CTG-CSA can be directly compiled into the standard EnergyPlus Weather (EPW) format. Note that the generated global horizontal irradiance (GHI) should be rigorously decomposed into Direct Normal Irradiance (DNI) and Diffuse Horizontal Irradiance (DHI) utilizing the Erbs empirical decomposition model before running EnergyPlus (.idf) simulations.
